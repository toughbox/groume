# 그루미(Groume) 앱 기획서

## 1. 프로젝트 개요

### 1.1 앱 명칭
**그루미(Groume)** - Group + Meeting의 합성어

### 1.2 앱 컨셉
이성간의 그룹 미팅을 매칭해주는 소셜 데이팅 플랫폼
- 1:1부터 대규모 그룹 미팅까지 다양한 인원 구성 지원
- 지역, 나이, 관심사 기반 스마트 매칭 시스템
- 미션 완료를 통한 게이미피케이션 요소 도입

## 2. 핵심 기능 명세

### 2.1 매칭 시스템
#### 2.1.1 매칭 프로세스
1. **미팅 신청 등록**
   - 그룹 구성원 정보 입력 (성별, 나이, 지역, 관심사)
   - 희망 미팅 조건 설정 (상대방 나이대, 지역)
   - 미팅 가능 일정 선택

2. **자동 매칭 알고리즘**
   - 인원수 매칭 (3:3, 5:5 등)
   - 지역 기반 매칭 (반경 설정 가능)
   - 나이대 매칭 (±3세 범위 조정 가능)
   - 관심사 유사도 분석

3. **매칭 확정 프로세스**
   - 양측 그룹 리더의 승인 필요
   - 미팅 성사 시 서로 연락할 수 있는 수단 공개(앱 내에서 채팅은 하지 않고 쪽지 등으로 연락.. 서버 비용 문제)
   - 미팅 장소 및 시간 조율 기능(아직 미정)

#### 2.1.2 매칭 조건 상세
- **필수 조건**: 인원수, 성별 구성, 지역
- **우선 조건**: 나이대, 미팅 목적(기본적으로는 이성 친구, 애인 만들기로 생각하고 있음)
- **부가 조건**: 직업, 취미

### 2.2 매칭 이용권 시스템
#### 2.2.1 이용권 획득 방법
1. **미션 완료**
   - 광고 시청 (30초 영상 광고) - 앱 오픈 후 광고 서비스 연결 후 진행
   - 미팅 후기 작성 및 인증
   - 친구 초대 (초대받은 친구 가입 시)
   - 일일 출석 체크

2. **구매**
   - 단건 구매: 1회 이용권
   - 패키지 구매: 5회, 10회 이용권
   - 월 정기 구독: 무제한 이용

#### 2.2.2 이용권 사용 규칙
- 미팅 신청 시 1개 소모
- 매칭 실패 시 이용권 환불 정책 생각해야함

## 3. 화면 구성 및 기능

### 3.1 인증 시스템
#### 3.1.1 로그인/회원가입
- **로그인**: 아이디/비밀번호 + JWT 토큰 발급
- **회원가입**: 기본 정보 + 프로필 사진 + 본인인증(sms 등의 과금 요소는 배제, 예: email 인증)
- **추후 확장**: 카카오, 네이버, 구글 소셜 로그인

### 3.2 메인 화면
#### 3.2.1 대시보드
- 보유 이용권 개수 표시
- 진행 중인 매칭 상태
- 완료된 미팅 히스토리
- 오늘의 미션 안내

### 3.3 미팅 신청 페이지
#### 3.3.1 그룹 구성
- 그룹 리더 정보 (본인)
- 멤버 추가 (최대 10명)
- 각 멤버별 기본 정보 입력

#### 3.3.2 매칭 조건 설정
- 희망 상대방 조건
- 미팅 가능 지역 선택
- 미팅 장소 (카페, 술집, 식당 등)
- 가능한 날짜/시간대

### 3.4 매칭 관리
#### 3.4.1 매칭 현황
- 대기 중인 신청
- 매칭된 상대방 정보

#### 3.4.2 미팅 관리
- 매칭 후 장소나 일정은 앱에서 관리 안함
- 미팅 후 평가 및 후기

### 3.5 마이페이지
#### 3.5.1 프로필 관리
- 개인 정보 수정
- 프로필 사진 관리
- 관심사 설정

#### 3.5.2 이용 내역
- 이용권 사용 히스토리
- 미팅 참여 기록
- 평점 및 후기 관리

## 4. 기술 스택 제안

### 4.1 Frontend
- **Framework**: React Native (크로스 플랫폼)
- **UI Library**: NativeBase(React Native Elements 는 보류, NativeBase 작업 시 마음에 들지 않을 경우 대체)
- **상태관리**: Redux Toolkit
- **네비게이션**: React Navigation

### 4.2 Backend
- **Runtime**: Node.js
- **Framework**: Express.js(NestJS는 일단 앱 규모가 작아 배제)
- **Database**: Mysql 커뮤니티 버전 (관계형) + Redis (캐싱)
- **인증**: JWT + Passport.js
- **파일 저장**: Minio S3

### 4.3 추가 서비스
- **푸시 알림**: Firebase Cloud Messaging
- **지도 서비스**: 카카오맵 API
- **결제**: 아임포트 또는 토스페이먼츠

## 5. 디자인 컨셉 제안

### 5.1 디자인 철학
**"Warm & Friendly Connection"**
- 따뜻하고 친근한 만남을 연상시키는 디자인
- 신뢰감과 안전함을 주는 UI/UX
- 젊고 활동적인 타겟층을 고려한 모던한 감성

### 5.2 컬러 팔레트
```
Primary Colors:
- Main: #FF6B6B (코랄 핑크) - 따뜻함과 열정
- Secondary: #4ECDC4 (민트 그린) - 신선함과 안정감
- Accent: #45B7D1 (스카이 블루) - 신뢰감과 소통

Neutral Colors:
- Background: #FAFAFA (라이트 그레이)
- Text Primary: #2C3E50 (다크 그레이)
- Text Secondary: #7F8C8D (미디엄 그레이)
- Border: #E8E8E8 (라이트 그레이)
```

### 5.3 UI 컴포넌트 라이브러리 선택
**추천: NativeBase v3**

#### 5.3.1 선택 이유
- React Native 전용 설계로 최적화된 성능
- 다크모드 지원 및 테마 커스터마이징 용이
- 접근성(Accessibility) 기본 지원
- TypeScript 완전 지원

#### 5.3.2 대안: React Native Elements
- 더 가벼운 라이브러리
- 커스터마이징 자유도가 높음
- 커뮤니티 지원이 활발

### 5.4 주요 화면별 디자인 가이드

#### 5.4.1 로그인 화면
```
- 상단: 앱 로고 + 캐치프레이즈
- 중앙: 로그인 폼 (카드 형태)
- 하단: 소셜 로그인 버튼들
- 배경: 그라데이션 또는 미팅 이미지
```

#### 5.4.2 메인 대시보드
```
- 헤더: 프로필 이미지 + 이용권 개수
- 카드 섹션: 진행 중인 매칭, 오늘의 미션
- 플로팅 버튼: 새 미팅 신청
- 하단 탭: 홈, 매칭, 마이페이지
```

#### 5.4.3 미팅 신청 화면
```
- 단계별 프로그레스 바
- 카드 기반 정보 입력 폼
- 이미지 업로드: 드래그 앤 드롭 지원
- 조건 설정: 슬라이더 및 토글 버튼 활용
```

### 5.5 아이콘 및 일러스트레이션
- **아이콘 스타일**: Outline 스타일 (Feather Icons 또는 Heroicons)
- **일러스트레이션**: 플랫 디자인 스타일의 캐릭터 및 상황 일러스트
- **이미지**: 실제 미팅 상황을 연상시키는 따뜻한 톤의 사진

### 5.6 애니메이션 및 인터랙션
- **페이지 전환**: 부드러운 슬라이드 애니메이션
- **버튼 피드백**: 탭 시 스케일 애니메이션
- **로딩**: 스켈레톤 UI 또는 커스텀 로딩 애니메이션
- **성공/실패**: 마이크로 인터랙션으로 피드백 제공

## 6. 개발 우선순위

### Phase 1 (MVP)
1. 사용자 인증 시스템
2. 기본 프로필 관리
3. 미팅 신청 기능
4. 단순 매칭 시스템

### Phase 2 (확장)
1. 고도화된 매칭 알고리즘
2. 이용권 시스템 및 미션
3. 결제 시스템
4. 푸시 알림
5. 미팅 후기 시스템

### Phase 3 (고도화)
1. 소셜 로그인
2. 실시간 위치 기반 매칭
3. AI 기반 추천 시스템
4. 이벤트 및 프로모션 시스템
