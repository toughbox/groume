# 프로젝트 개발 규칙

## 라이브러리 관리 규칙

### 1. 라이브러리 교체 금지
- **사용자 허락 없이 기존 라이브러리를 다른 라이브러리로 전면 교체하지 않습니다**
- 예시: NativeBase → React Native Paper 같은 UI 라이브러리 교체 금지
- 버전 다운그레이드/업그레이드는 허용

### 2. 라이브러리 추가
- 필요에 의한 새로운 라이브러리 추가는 가능합니다
- 추가 전 사용자에게 목적과 필요성을 설명해야 합니다

### 3. 문제 해결 우선순위
1. 설정 파일 수정 (config, babel 등)
2. 코드 레벨 수정
3. 라이브러리 버전 조정
4. **마지막 수단**: 라이브러리 교체 (사용자 허락 필요)

## 커뮤니케이션 규칙

### 1. 이모지 사용 제한
- 과도한 이모지 사용을 지양합니다
- 필요한 경우에만 적절히 사용합니다
- 전문적이고 간결한 메시지를 유지합니다

### 2. 응답 스타일
- 명확하고 구체적인 설명을 제공합니다
- 문제 해결 과정을 투명하게 공유합니다
- 불필요한 장식적 표현을 피합니다

## 개발 원칙

### 1. 기존 아키텍처 존중
- 현재 프로젝트 구조와 설계를 최대한 유지합니다
- 변경 시 반드시 이유를 명시합니다

### 2. 최소 침입적 해결
- 문제 해결 시 가장 적은 변경으로 해결을 시도합니다
- 전면적인 변경보다는 점진적인 개선을 추구합니다

### 3. 사용자 중심
- 모든 결정은 사용자의 승인을 받습니다
- 대안이 있을 경우 선택권을 제공합니다

### 4. 폴더, 파일명에 한눈에 알아볼 수 없는 의미 사용하지 않기
- 예: LoginScreenSimple.tsx 의 Simple은 의미 없음

이 규칙들은 프로젝트 개발 과정에서 준수되어야 하며, 필요시 사용자와 협의하여 수정할 수 있습니다.
